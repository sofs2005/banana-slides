## ✨ Project Origin

Have you ever found yourself in this predicament: your presentation is due tomorrow, but the slides are still blank; your mind is full of brilliant ideas, yet all your enthusiasm is drained by tedious layout and design work?

We long to quickly create professional and well-designed presentations. While traditional AI PPT generation apps largely meet the need for "speed," several issues persist:

- 1️⃣ Limited to preset templates, with no flexibility to adjust styles
- 2️⃣ Low degree of freedom, making multi-round revisions difficult
- 3️⃣ Similar-looking outputs with severe homogenization
- 4️⃣ Low-quality assets that lack specificity
- 5️⃣ Disconnected text-and-image layouts with a poor sense of design

These shortcomings make it difficult for traditional AI PPT generators to simultaneously satisfy our dual demands for "speed" and "aesthetics." Even those claiming to be "Vibe PPT" are, in my eyes, still far from being "Vibe" enough.

However, the emergence of the nano banana🍌 model has changed everything. I tried using 🍌pro to generate PPT pages and found that the results were exceptional in terms of quality, aesthetics, and consistency. It can accurately render almost all text requested in the prompt while strictly following the style of reference images. So, why not build a native "Vibe PPT" application based on 🍌pro?

## 👨‍💻 Use Cases

1. **Beginners**: Quickly generate beautiful PPTs with zero barrier and no design experience required, reducing the hassle of selecting templates.
2. **PPT Professionals**: Gain design inspiration quickly by referring to AI-generated layouts and combinations of text and graphic elements.
3. **Educators**: Rapidly convert teaching content into illustrated lesson plan PPTs to improve classroom effectiveness.
4. **Students**: Quickly complete assignment presentations and focus energy on content rather than formatting and styling.
5. **Business Professionals**: Rapidly visualize business proposals and product introductions, with quick adaptation to multiple scenarios.

  <b>🎯 Goal: Lower the barrier to PPT creation, enabling everyone to quickly create aesthetic and professional presentations</b>

## 🎯 Features

### 1. Flexible and Diverse Creative Paths

Supports three starting modes—**Ideas**, **Outlines**, and **Page Descriptions**—to accommodate different creative workflows.
- **One-sentence Generation**: Enter a topic, and AI automatically generates a well-structured outline and page-by-page content descriptions.
- **Natural Language Editing**: Supports modifying outlines or descriptions via natural language (e.g., "Change page three to a case study"), with real-time AI adjustments.
- **Outline/Description Mode**: Supports both one-click batch generation and manual fine-tuning of details.

<img width="2000" height="1125" alt="image" src="https://github.com/user-attachments/assets/7fc1ecc6-433d-4157-b4ca-95fcebac66ba" />

### 2. Powerful Asset Parsing Capabilities

- **Multi-format Support**: Upload PDF, Docx, MD, Txt, and other files; the system automatically parses the content in the background.
- **Intelligent Extraction**: Automatically identify key points, image links, and chart information from the text, providing rich material for generation.
- **Style Reference**: Support for uploading reference images or templates to customize the PPT style.

No longer restricted by complex menus, issue modification commands directly through **natural language**.
- **Partial Redraw**: Make verbal modifications to unsatisfactory areas (e.g., "change this chart to a pie chart").
- **Full-page Optimization**: Generate high-definition, stylistically consistent pages based on nano banana pro🍌.

# Lobe Chat - Database Migration Guide

This guide will help you migrate Lobe Chat data from local browser storage (IndexedDB) to the database version (PostgreSQL).

## 1. Preparation

Before you begin, please ensure you have completed the following:

- [x] Back up your data (Export configuration in Settings -> General -> Backup & Restore)
- [x] Install and configure Docker and Docker Compose
- [x] Prepare a PostgreSQL database instance (you can use Docker deployment or a managed service)
- [x] Prepare an S3-compatible object storage service (for storing images and other files)
- [x] Prepare an authentication service (e.g., Clerk or NextAuth)

## 2. Deploying the Database Version

You can refer to the [Self-hosted Database Version Deployment Guide](https://github.com/lobehub/lobe-chat/wiki/Deployment-Database) to deploy your Lobe Chat database version.

Main steps include:

1.  Configuring environment variables (`.env` file)
2.  Starting services with Docker Compose
3.  Verifying if the service is running normally

## 3. Data Migration Process

Due to significant architectural differences between the local storage version and the database version, we currently recommend the following migration method:

### 3.1 Export Data

1.  Open the Lobe Chat you are currently using (local storage version)
2.  Go to `Settings` -> `General` -> `Backup & Restore`
3.  Click `Export Configuration` and save the exported `.json` file

### 3.2 Import Data

1.  Open the newly deployed Lobe Chat (database version)
2.  Complete the login process
3.  Go to `Settings` -> `General` -> `Backup & Restore`
4.  Click `Import Configuration` and select the `.json` file you just exported

## 4. Considerations

- **File Migration**: Images and other files in the local storage version are stored in the browser's IndexedDB and will be converted to Base64 encoding when exporting to JSON. After importing to the database version, these files will be automatically uploaded to your configured S3 storage.
- **Data Volume Limits**: If your data volume is very large (e.g., thousands of messages), the JSON file exported by the browser may be very large. Please keep the browser page open and wait patiently during the import process.
- **Multi-device Sync**: Once migrated to the database version, you will enjoy a native cross-device synchronization experience without the need for manual export/import.

## 5. Frequently Asked Questions (FAQ)

### Q: Will the original local data still exist after migration?
A: Yes. The import operation will not delete the data in your local browser; it only copies the data into the database.

### Q: Can I migrate back from the database version to the local storage version?
A: Theoretically yes, you can use the export/import function in the same way. However, we recommend using the database version for better performance and stability.

## 6. Environment Variable Reference

When deploying the database version, you need to configure the following key environment variables:

## **🔧 FAQ**

1. **Garbled or blurry text on generated pages**
    - You can select a higher output resolution (the OpenAI format may not support resolution adjustments; Gemini format is recommended). Based on testing, increasing the resolution from 1k to 2k before generating the page significantly improves text rendering quality.
    - Please ensure that the specific text content to be rendered is included in the page description.

2. **Poor results when exporting editable PPTs, such as overlapping text or missing styles**
    - In 90% of cases, this is due to an API configuration issue. Please refer to the troubleshooting and solutions in [issue 121](https://github.com/Anionex/banana-slides/issues/121).

3. **Does it support the free tier Gemini API Key?**
    - The free tier only supports text generation and does not support image generation.

4. **503 Error or Retry Error occurs during content generation**
    - You can check the Docker backend logs using the commands in the README to locate the detailed error for the 503 issue. This is usually caused by incorrect model configuration.

5. **Why doesn't the API Key take effect after setting it in .env?**
    - The Docker container must be restarted after editing the `.env` file during runtime to apply the changes.
    - If parameters were previously configured on the web settings page, they will override the parameters in `.env`. You can restore the `.env` settings by clicking "Restore Default Settings".
